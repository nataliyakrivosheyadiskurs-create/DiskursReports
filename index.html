Разделы приложения
Неделя

Главный рабочий раздел. Показывает данные за прошедшие 7 дней — сколько потрачено, сколько лидов получено и по какой цене.

Период — по умолчанию прошедшая неделя. Можно выбрать «Произвольный период» и задать любые даты.
Фильтры — можно отфильтровать данные по конкретной кампании или группе объявлений.
Сравнение — рядом с цифрами показывается изменение в % относительно предыдущего такого же периода. Зелёный — улучшение, красный — ухудшение.
Топ-3 креатива — лучшие объявления по количеству результатов и по самой низкой цене за лид.
Картинки — нажмите на любое изображение чтобы увеличить его.
PDF — кнопка «PDF» в правом верхнем углу. Можно выбрать какие разделы включить в отчёт для клиента.
Месяц

Статистика по каждой кампании с разбивкой по дням. Удобно чтобы видеть динамику — в какие дни было больше лидов, когда росли затраты. Выберите нужный месяц из списка и нажмите «Загрузить».

Год

Сводная таблица по кампаниям с разбивкой по месяцам с начала года. Видно сколько потрачено в каждом месяце по каждой кампании — удобно для квартальных отчётов и планирования бюджета.

Проекты

Каждый проект — это один клиент. У одного клиента может быть несколько рекламных кабинетов — данные из них суммируются.

Название — например «Yves Ponroy» или «Sante Group»
Рекламные кабинеты — ID вида act_XXXXXXXXX. Можно добавить несколько.
Сквозная аналитика — подключается если у клиента есть CRM. Необязательно.
Цели кампаний

Очень важный раздел. Здесь вы говорите приложению что считать результатом в каждой кампании.

⚠️ Если цели не заданы — в отчёте будет 0 результатов по всем кампаниям. Обязательно заполните этот раздел для каждого нового проекта.
Переписки — реклама на WhatsApp или Messenger. Считаются начатые диалоги.
Лид-форма — реклама с формой заявки внутри Facebook. Считаются заполненные формы.
Пиксельное событие — реклама на сайт с пикселем. Считаются события Lead на сайте.
Охват / Просмотры — имиджевая реклама. Лиды не считаются, показываются только затраты и показы.
Настройки

Инструкция по подключению CRM
Пошаговое руководство — без технических знаний
Зачем это нужно?

Когда реклама работает на переписки или заявки — Facebook считает все входящие обращения. Но не все они реально качественные. Подключив CRM, вы увидите сколько из этих обращений стали квалифицированными лидами — то есть людьми, которые действительно заинтересованы в покупке. Это позволяет оценивать рекламу честно, а не по количеству кликов.

1
Убедитесь что UTM-метки настроены правильно

UTM-метки — это специальные пометки в ссылках, которые помогают понять откуда пришёл клиент. Без них невозможно связать данные из рекламы с данными из CRM.

В рекламном кабинете Facebook должны быть заданы:
utm_source Всегда пишите fb — это означает что клиент пришёл из Facebook
utm_campaign Используйте {{campaign.name}} — Facebook сам подставит название кампании
utm_content Используйте {{adset.name}}_{{ad.name}} — подставится группа и объявление
💡 Проверить можно так: зайдите в Битрикс24, откройте любую сделку которая пришла с рекламы, и посмотрите заполнено ли поле UTM Source. Если там написано fb — всё настроено правильно.
2
Выгрузите данные из Битрикс24 в Google Таблицу

Нам нужна таблица со сделками из Битрикс24. В ней должны быть колонки: дата создания, UTM-метки и статус/квалификация лида. Это делается один раз — потом таблица обновляется автоматически.

а
Попросите разработчика или интегратора Битрикс24 настроить экспорт сделок в Google Таблицу. Им нужно передать: нужные поля (дата, UTM Source, UTM Campaign, UTM Content, стадия, квалификация), формат выгрузки — Google Sheets, частота — ежедневно.
б
Если Битрикс настраивали вы сами — выгрузку можно сделать через раздел CRM → Сделки → Экспорт, или через встроенный отчёт с сохранением в Google Sheets.
в
В эту же таблицу нужно добавить лист с данными Facebook — обычно он уже есть если вы используете нашу систему выгрузки.
3
Настройте проект в приложении
1
Перейдите в раздел Проекты и откройте нужный проект (или создайте новый).
2
В разделе Сквозная аналитика выберите CRM — Битрикс24 или AmoCRM.
3
Введите ID Google таблицы. Это длинный код в адресе таблицы — между /d/ и /edit. Например: 1qgsq8cdsWuVDKP3qjbJN0e7Uqcid...
4
Укажите название листов — точно как в таблице (регистр важен). Например: Bitrix и Facebook 2025.
5
Укажите метод квалификации и значения. Если в колонке "Квалификация" у хороших лидов написано BBB — выберите "По значению" и впишите BBB. Если используете систему ABCD — выберите соответствующий пункт.
6
Нажмите Сохранить. Появится окно с инструкцией — скопируйте код из него и передайте таргетологу или разработчику.
4
Запустите синхронизацию (делает таргетолог)

Этот шаг делает таргетолог или человек который настраивал выгрузку данных. Если вы боитесь кода — просто передайте ему эту инструкцию.

1
Откройте Google Таблицу со сквозной аналитикой → меню Расширения → Apps Script.
2
Нажмите + рядом с "Файлы" → создайте файл с названием crm_sync.
3
Вставьте код из файла crm_sync.gs и код который появился после сохранения проекта.
4
Выберите функцию RUN_SYNC_CRM и нажмите кнопку ▶ Запустить. При первом запуске Google попросит разрешения — нажмите "Разрешить".
5
Настройте автозапуск: Триггеры (значок часов) → Добавить триггер → функция RUN_SYNC_CRM → Ежедневно → 06:00. После этого данные будут обновляться каждое утро автоматически.

Частые вопросы
Данные в отчёте не совпадают с тем что я вижу в Битрикс
Скорее всего не все сделки имеют UTM-метки. Это происходит когда клиент написал напрямую (не по рекламной ссылке), или когда UTM не были настроены в момент запуска рекламы. Это нормально — в отчёте показываются только те лиды которые пришли с рекламы.
Когда обновляются данные?
Если настроен автозапуск — каждое утро в 06:00. Данные за вчерашний день появятся утром. Если нужно обновить прямо сейчас — попросите таргетолога запустить синхронизацию вручную.
Можно ли использовать AmoCRM вместо Битрикс?
Да. Схема та же — нужна выгрузка сделок с UTM-метками в Google Таблицу. Выберите AmoCRM при настройке проекта.
Что такое квалифицированный лид?
Это человек который проявил реальный интерес к покупке — прошёл первичный отбор менеджером по продажам. В разных компаниях это определяется по-разному: по значению в специальном поле (BBB, A, B), по стадии воронки, или по системе оценки ABCD. Вы настраиваете правила квалификации при создании проекта.
